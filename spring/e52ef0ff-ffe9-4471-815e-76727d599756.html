Spring Boot 基础

简介

Spring Boot 是 Spring 家族中的一个全新的框架，它用来简化 Spring 应用程序的创建和开发过程，也可以说 Spring Boot 能简化我们之前采用 Spring + SpringMVC + MyBatis 框架进行开发的过程。

在以往我们采用 Spring + SpringMVC + MyBatis 框架进行开发的时候，搭建和整合三大框架，我们需要做很多工作，比如配置 web.xml，配置 MyBatis，并将它们整合在一起等，而 Spring Boot 框架对此开发过程进行了革命性的颠覆，完全抛弃了繁琐的 xml 配置过程，采用大量的默认配置简化开发过程。

所以采用 Spring Boot 可以非常容易和快速地创建基于 Spring 框架的应用程序，它让编码、配置、部署及监控变得简单了。正因为 Spring Boot 它化繁为简，让开发变得极其简单和快速，所以在业界备受关注。

Spring Boot 特点

  • 快速构建项目，为基于 Spring 的开发提供更快的入门体验
  • 对主流开发框架的无配置集成
  • 项目可独立运行，无需外部依赖 Servlet 容器
  • 提供了一些大型项目中常见的非功能性特性，如嵌入式服务器、安全指标、健康检测
  • 极大地提高了开发、部署效率
  • 与云计算的天然集成

Spring Initializr

我们可以使用 Spring 官网 上自带的模版（Spring Initializr）来创建 Spring Boot 应用程序，也可以使用 IDEA 上的 Spring Initializr 来创建。这种方法需要从 Spring 官网上下载模版包，需要网络，同时国内访问外网比较慢，因此这种方式不推荐使用。

配置

Spring Boot 是约定优于配置的，所以其中很多配置都是有默认值的。但如果想使用自己的配置替换默认配置的话，就可以使用 application.properties 或 application.yml 进行配置 (推荐使用 application.yml)

YML 文件是 YAML (YAML Aint Markup Language) 文件格式，YAML 是一种直观的能够被电脑识别的数据序列化格式，并且容易被人类阅读，容易和脚本语言交互的，可以被支持 YAML 库的不同的编程语言程序导入，比如： C/C++, Ruby, Python, Java, Perl, C#, PHP 等。YML 文件是以数据为核心的，比传统的 xml 文件更加简洁。

Spring Boot 所有默认配置在 Spring 官方文档中 列出。注意：官方为了方便，使用 properties 的格式列出所有配置，我们可以将其改为 yaml 格式。
