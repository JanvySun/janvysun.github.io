Logstash 的使用

发表于2023-01-02|更新于2023-01-02|Software
|字数总计:3.1k|阅读时长:15分钟|阅读量:

前言

本文所采用的 Logstash 版本为 6.8.4，新版 Logstash 配置可能会有所不同，详情请参考 Logstash 官方文档

下载链接：https://artifacts.elastic.co/downloads/logstash/logstash-6.8.4.tar.gz

7 及后续版本下载链接：https://artifacts.elastic.co/downloads/logstash/logstash-7.17.9-linux-x86_64.tar.gz


启动

tar 包下载完之后，解压，然后进入目录中，运行 bin/logstash -f config/logstash-sample.conf 命令即可启动 logstash

比较运算符

  • 相等：==、!=、<、>、<=、>=
  • 正则：=~(匹配正则)、!~(不匹配正则)
  • 包含：in(包含)、not in(不包含)

布尔运算符

  • and（与）
  • or（或）
  • nand（非与）
  • xor（非或）

一元运算符

  • !（取反）
  • ()（复合表达式）
  • !()（对复合表达式的结果取反）

date

使用 date 插件解析字段中的日期，然后使用该日期或时间戳作为事件的 logstash 时间戳(@timestamp)。对于排序事件和导入旧数据，日期过滤器尤其重要。如果您在事件中没有得到正确的日期，那么稍后搜索它们可能会出现问题。

如果时间戳尚未在事件中设置，logstash 将根据第一次看到事件(在输入时)创建一个时间戳。对于文件输入，时间戳设置为每次读取的时间。

mutate

对字段做处理 重命名、删除、替换和修改字段。

  • covert：类型转换。类型包括：integer，float，integer_eu，float_eu，string和boolean

  • split：使用分隔符把字符串分割成数组

  • merge：合并字段。数组和字符串，字符串和字符串

  • rename：对字段重命名

  • join：用分隔符连接数组，如果不是数组则不做处理

  • gsub：用正则或者字符串替换字段值。仅对字符串有效　

  • update：更新字段。如果字段不存在，则不做处理

  • replace：更新字段。如果字段不存在，则创建

  • ruby：ruby插件可以执行任意Ruby代码


output

输出插件，将事件发送到特定目标。

stdout

标准输出。将事件输出到屏幕上

file

将事件写入文件

kafka

将事件发送到kafka

elasticseach

在 es 中存储日志
