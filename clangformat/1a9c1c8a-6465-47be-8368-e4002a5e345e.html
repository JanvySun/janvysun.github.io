Clion 使用 ClangFormat

Clion 需要在设置中开启此功能才能使用 ClangFormat 格式化代码：

在设置中的 Editor -> Code Style -> General -> Enable ClangFormat

配置项

BasedOnStyle

string 类型，用于设置基于的编码规范, 可选值如下：

  • LLVM: https://llvm.org/docs/CodingStandards.html
  • Google: https://google.github.io/styleguide/cppguide.html
  • Chromium: https://chromium.googlesource.com/chromium/src/+/refs/heads/main/styleguide/styleguide.md
  • Mozilla: https://firefox-source-docs.mozilla.org/code-quality/coding-style/index.html
  • WebKit: https://www.webkit.org/coding/coding-style.html
  • Microsoft: https://docs.microsoft.com/en-us/visualstudio/ide/editorconfig-code-style-settings-reference
  • GNU: https://www.gnu.org/prep/standards/standards.html
  • InheritParentConfig: 继承父目录的编码规范, 如果有的话, 不是一个真正的编码规范
  • None: 不使用, 即自动配置, 也就是本文件中的自定义内容

C++/C 语言推荐使用 Microsoft

Language

设置语言，可针对不同语言设置不同的策略，如下：

AccessModifierOffset

Integer 类型，用于设置访问声明符缩进，如下：

AlignAfterOpenBracket

boolean/string 类型，用于设置开括号后的对齐(包括小括号/大括号/尖括号), 建议使用 Align

配置项如下：

  • Align: 对齐开括号，即在换行情况下，换行的参数跟开括号对齐，建议使用
  • DontAlign: 不对齐开括号，即换行时使用 ContinuationIndentWidth 配置的空格数
  • AlwaysBreak: 永远换行，即第一个参数都不允许粘连括号，会强制换行，换行后使用配置空格数对齐
  • BlockIndent: 同 AlwaysBreak，多了一个操作：如果参数不固定在同一行，闭括号将在下一行

AlignArrayOfStructures

string 类型，用于设置结构体数组统一初始化对齐, 建议配置 Left

配置项如下：

  • None: 不做处理, 即保留开发者的代码
  • Left: 左对齐
  • Right: 右对齐

AlignConsecutiveAssignments

Object 或 string 类型，用于设置连续赋值语句的对齐，即多个赋值语句连续出现时的对齐策略配置, 包含多个子配置项，建议配置为 None

Object

可对其子项一一配置，如下：

string

当配置为 string 时，则使用内置好的配置：

  • None：所有项均为 false
  • Consecutive: 仅设置 Enabled 为 true
  • AcrossEmptyLines: 设置子项 AcrossEmptyLines 为 true
  • AcrossComments: 设置子项 AcrossComments 为 true
  • AcrossEmptyLinesAndComments: 设置 AcrossEmptyLines 及 AcrossComments

AlignConsecutiveBitFields

Object 或 string 类型，同 AlignConsecutiveAssignments，表示连续位定义语句出现时，是否需要对齐 : 符号，位变量定义用得少，建议为 None

AlignConsecutiveDeclarations

Object/string 类型，用于设置是否对齐连续声明，配置项同上，建议为 None

AlignConsecutiveMacros

Object/string 类型，用于设置是否对齐宏定义，配置项同上，建议为 Consecutive

AlignConsecutiveShortCaseStatements

object, 始于 17 版本，设置连续短 case 对齐

AlignEscapedNewlines

string 类型，用于设置续行符 \ 对齐，建议设置为 Left

  • DontAlign: 不做操作
  • Left: 尽可能向左对齐, 即最长一行代码为准
  • Right: 将续行符写在最远处（ColumnLimit 定义的位置）

AlignOperands

string 类型，在二元/一元表达式中的操作数对齐，建议设置 Align

配置项如下：

  • DontAlign: 不做对齐, 在操作数换行后, 将使用 ContinuationIndentWidth 来对齐
  • Align: 即换行时, 操作数(or 操作符加操作数)跟上一行的第一个操作数左对齐, 具体操作符要不要换行, 由BreakBeforeBinaryOperators配置决定

AlignTrailingComments

string 类型，设置连续行后注释的对齐，建议 Always

AllowAllArgumentsOnNextLine

boolean, 是否允许所有参数都在下一行

AllowAllParametersOfDeclarationOnNextLine

AllowShortBlocksOnASingleLine

string 类型，设置短的 Blocks 语句是否放在同一行，建议：Empty

配置项：

  • Never: 从不
  • Empty: 仅合并空的 Blocks
  • Always: 空与短 Blocks 都将合并

AllowShortCaseLabelsOnASingleLine

boolean 类型，设置短 case 语句是否合并，建议 true

AllowShortEnumsOnASingleLine

boolean 类型，设置短 Enums 是否合并至一行中，建议 false

AllowShortFunctionsOnASingleLine

string 类型，设置是否允许短的函数放在同一行，建议：Empty

配置项：

  • None: 从不
  • InlineOnly: 定义在类中的短函数
  • Empty: 空函数
  • Inline: 定义在类中的空函数
  • All: 所有短函数都将合并

AllowShortIfStatementsOnASingleLine

string 类型，设置是否将简单的if(else/else if)语句中的body跟if(else/else if)放置于同一行

配置项：

  • Never: 永远不
  • WithoutElse: 没有 else/else if 时允许
  • OnlyFirstIf: 只有第一个 if 允许
  • AllIfAndElse: 所有的 if/else 都允许

AllowShortLambdasOnASingleLine

string 类型，设置是否允许 lambda 在一行中，配置项同上，建议 All

AllowShortLoopsOnASingleLine

boolean 类型， 设置是否允许短 loop 语句体跟 loop 语句共行, true/false, 建议false

AlwaysBreakAfterReturnType

string 类型，设置函数类型后面是否换行，建议：None

AlwaysBreakBeforeMultilineStrings

boolean 类型，true/false, 建议false

AlwaysBreakTemplateDeclarations

string 类型，设置模板声明换行风格，建议 Yes

  • No: 永远不对开发者的风格作处理
  • MultiLine: 即仅在开发者写的模板声明(包括函数)跨越多行时, 进行换行, 否则维持原样
  • Yes: 不管如何都进行换行

AttributeMacros

string 数组，设置属性宏列表, 自定义, 用于语言扩展或静态分析注解（暂不知有何作用）

BinPackArguments

boolean类型，设置函数调用时的参数(Arguments)是否放置于一行, false 不放置, true 强制一个调用参数一行, 建议 false

BinPackParameters

boolean 类型，设置函数定义参数(Parameters)是否放置于一行, 同 BinPackArguments

BitFieldColonSpacing

string 类型，设置位定义:前后是否增加空格

配置项：

  • Both: 前后都添加
  • Before: 只在前增加
  • After: 只在后增加
  • None: 不增加，建议，没有必要因为过多的空格影响代码紧凑

BraceWrapping

object 类型，设置大括号换行风格

注意：在使用此配置时，需要将 BreakBeforeBraces 设置为 Custom：

配置子项如下：

  • AfterCaseLabel: true/false，在 case 后的大括号是否换行
  • AfterClass: true/false，class后
  • AfterControlStatement: Always/Never/MultiLine，控制语句(if/for/while/switch/…)后是否换行
    • Never: 永远不, 即永远将语句体的大括号放置于控制语句同一行
    • MultiLine: 多行控制语句才进行换行
    • Always: 永远换行, 建议
  • AfterEnum: true/false
  • AfterFunction: true/false
  • AfterNamespace: true/false
  • AfterObjCDeclaration: true/false
  • AfterStruct: true/false
  • AfterUnion: true/false
  • AfterExternBlock: true/false
  • BeforeCatch: false/true
  • BeforeElse: true/false
  • BeforeLambdaBody: false/true
  • BeforeWhile: false/true
  • IndentBraces: false/true
  • SplitEmptyFunction: true/false
  • SplitEmptyRecord: true/false
  • SplitEmptyNamespace: true/false

BracedInitializerIndentWidth

unsigned 类型，设置数组初始化时，大括号中值的缩进量，建议值：4

BreakBeforeBinaryOperators

string，设置二元操作符前是否换行, 建议为 None

BreakBeforeBraces

string，大括号换行风格，建议 Linux 风格，也可使用 Custom，然后用 BraceWrapping 自定义

BreakBeforeConceptDeclarations

string，concept 声明前是否换行, 建议 Always

BreakBeforeTernaryOperators

boolean，三元表达式换行样式，建议 true

BreakConstructorInitializers

string, 构造函数初始化列表换行风格, 使用 AfterColon 适合

BreakInheritanceList

string, 继承列表换行风格，建议 AfterColon

BreakStringLiterals

boolean, 字符串是否换行, true

ColumnLimit

设置代码列字符上限

CommentPragmas

pragma注释，暂时未知效果

CompactNamespaces

boolean, 是否压缩 namespace，建议 false

ConstructorInitializerIndentWidth

构造函数初始化列表缩进, 建议 0

ContinuationIndentWidth

续行对齐缩进, 建议与缩进保持一致

Cpp11BracedListStyle

boolean, 设置 C++11 的统一初始化列表大括号风格, 建议 false

DerivePointerAlignment

boolean，是否开启文件分析, 根据文件中的*/&使用情况更新clang-format设定, 在无法决定时, 使用PointerAlignment代替, 不建议开启

DisableFormat

是否禁用 format，建议 false

EmptyLineAfterAccessModifier

string, 访问限定后是否添加空行, 建议Never

EmptyLineBeforeAccessModifier

string, 访问限定前是否要求空行, 建议LogicalBlock

ExperimentalAutoDetectBinPacking

实验性的自动检测同行并进行操作，建议 false

FixNamespaceComments

是否强制在 namespace 结尾增加注释 // namespace xxx，建议为 true

ForEachMacros

Macro宏

IfMacros

If宏

IncludeBlocks

include代码块操作, 前提是SortIncludes开启:

  • Preserve: 只对每个代码块排序
  • Merge: 对所有代码块合并, 并在合并后排序
  • Regroup: 对所有include块进行分析, 并重新分块, 不建议!

IncludeCategories

Include种类, 默认即可

IncludeIsMainRegex

IncludeIsMainSourceRegex

IndentAccessModifiers

boolean, 是否缩进访问控制

IndentCaseBlocks

是否缩进case body，建议 false

IndentCaseLabels

是否缩进case语句, 建议false

IndentExternBlock

extern “C”缩进, 建议 AfterExternBlock

IndentGotoLabels

是否缩进goto标签，建议 true

IndentPPDirectives

预处理指示(PPD-PreProcessor Directive)缩进, 建议None

  • None: 不缩进
  • AfterHash: #不缩进, #后面的指示缩进
  • BeforeHash: #跟前缩进

IndentRequiresClause

设置模板 require 是否缩进，建议 true

IndentWidth

设置缩进宽度，建议 4

IndentWrappedFunctionNames

函数名换行时, 是否缩进(即返回值跟名字不同行时), 建议 false

InsertBraces

是否在代码块中(if/else/for/do/while)强制插入大括号, 建议 false

InsertTrailingCommas

是否强制插入拖尾的,, 建议为 None

KeepEmptyLinesAtTheStartOfBlocks

是否保留代码块开始处的空行，建议 false

LambdaBodyIndentation

Lambda 表达式 body 缩进样式，建议 Signature

LineEnding

Line ending style (\n or \r\n) to use.

MaxEmptyLinesToKeep

空行保持, 建议为1

NamespaceIndentation

namespace内的对齐, 直接使用None即可, 即所有namespace内(包括内嵌的)都不indent

PPIndentWidth

预处理对齐宽度，建议 -1

PackConstructorInitializers

是否打包构造函数初始化列表, 建议 Never, 可选:

  • Never: 永远不做操作, 即一个参数一行
  • BinPack: 两个参数一行
  • CurrentLine: 所有参数放置于一行, 如果放不下, 就一个参数一行
  • NextLine: 同CurrentLine有点像, 唯一不同就是如果放不行, 将剩余参数放置于下一行(即不自动一参一行)

PointerAlignment

指针对齐, 建议Right

QualifierAlignment

注释关键字对齐(const/volatile), 建议Leave

  • Leave: - 不改变开发者定义
  • Left: 位于类型前
  • Right: 位于类型后
  • Custom: 自定义，使用 QualifierOrder 项来定义先后次序

QualifierOrder

定义Qualifier先后次序(注意：必须包含 type):

  • const
  • inline
  • static
  • friend
  • constexpr
  • volatile
  • restrict
  • type

ReferenceAlignment

引用对齐, 可选:

  • Pointer: 使用 ‘PointerAlignment’ 的配置, 建议使用
  • Left: Left
  • Right: Right

ReflowComments

是否允许 clang-format 尝试重新粘合注释(true/false), 建议 false

RemoveBracesLLVM

是否移除多余的 {}, false

RemoveParentheses

是否移除多余的 (), false

RemoveSemicolon

是否移除多余的分号，建议 true

RequiresClausePosition

模板中的 require 语句位置, 建议OwnLine

SeparateDefinitionBlocks

分隔不同定义块, 建议Always, 可选:

  • Leave - 不处理, 建议, 即由业务决定, 也可以使用Always
  • Always - 永远进行分隔
  • Never: 永远 不进行, 不建议

ShortNamespaceLines

大于多少行namespace内的代码行时才在namespace结尾添加 // namespace xxx, 建议0，即无论如何都添加

SortIncludes

对 includes 排序选项, 可选:

  • Never: 永远不, 建议
  • CaseSensitive: 大小写敏感排序
  • CaseInsensitive: 大小写不敏感排序

SpaceAfterCStyleCast

C 风格 cast的类型括号后面是否增加 space(true/false), 建议false

SpaceAfterLogicalNot

非操作(!)后面是否加space(true/false), 建议false

SpaceAfterTemplateKeyword

template关键字后面是否加 space(true/false), 建议false

SpaceAroundPointerQualifiers

指针修饰的space添加, 建议 Default, 即使用PointerAlignment代替

SpaceBeforeAssignmentOperators

赋值语句操作符前是否添加space(true/false), 建议true

SpaceBeforeCaseColon

case 语句:前是否增加space(true/false), 建议false

SpaceBeforeCpp11BracedList

c++11的统一初始化列表的大括号中是否添加space(true/false), 建议 true

SpaceBeforeCtorInitializerColon

构造函数初始化列表:前是否加space(true/false), 建议false

SpaceBeforeInheritanceColon

继承列表的 : 前是否加space(true/false), 建议true

SpaceBeforeParens

圆括号前是否增加空格，建议只在控制语句的圆括号前增加, 即配置为ControlStatements即可

SpaceBeforeParensOptions

自定义圆括号前是否添加空格，SpaceBeforeParens 为 Custom 时使用

SpaceBeforeRangeBasedForLoopColon

Loop 关键字前前是否增加 space, 建议true

SpaceBeforeSquareBrackets

中括号前是否加space, false

SpaceInEmptyBlock

大括号内容为空时是否添加 space, 建议false

SpacesBeforeTrailingComments

注释前的空格数, 建议1

SpacesInAngles

<>里面是否增加space, 不建议, 配置成Never即可

SpacesInContainerLiterals

容器初始化列表[]/{}里面是否增加 space, 不建议(跟C++11风格保持一致)

SpacesInLineCommentPrefix

行注释前的空格范围数量, 建议Maximum关闭, 设置成-1, 即//到你的注释内容前的空格数量至少是1, 至多是无穷
