Redis 的基本使用

发表于 2020-04-23 更新于 2021-06-30 分类于 Database 阅读次数：
本文字数： 5.4k 阅读时长 ≈ 11 分钟

Redis 是一个高性能的 key-value 数据库，属于 NoSQL 类型的数据库。本文介绍其基本使用。

Redis 特性

Redis 与其他 key - value 缓存产品有以下三个特点：

  • Redis支持数据的持久化，可以将内存中的数据保存在磁盘中，重启的时候可以再次加载进行使用。
  • Redis不仅仅支持简单的key-value类型的数据，同时还提供list，set，zset，hash等数据结构的存储。
  • Redis支持数据的备份，即master-slave模式的数据备份。

Redis 优势：

  • 性能极高: Redis能读的速度是110000次/s,写的速度是81000次/s 。
  • 丰富的数据类型: Redis支持二进制案例的 Strings, Lists, Hashes, Sets 及 Ordered Sets 数据类型操作。
  • 原子: Redis的所有操作都是原子性的，同时Redis还支持对几个操作全并后的原子性执行。
  • 丰富的特性: Redis还支持 publish/subscribe, 通知, key 过期等等特性。

安装

Redis 的安装非常方便，只需要在 官网 下载压缩包解压然后 make 即可。

  • step1:下载之后解压
    tar -zxvf redis-3.2.8.tar.gz

  • step2:进⼊⽬录
    直接使用 make 命令进行编译
    然后使用 make test 命令进行测试 (需要的时间可能比较长)
    最后使用 make install 命令进行安装 (会安装到/usr/local/bin目录下)

  • step3:安装完成后，我们进入目录 /usr/local/bin 中查看
    cd /usr/local/bin
    ls -alh redis*

    其中：
    redis-server:redis服务器
    redis-cli:redis命令行客户端
    redis-benchmark:redis性能测试工具
    redis-check-aof:AOF文件修复工具
    redis-check-rdb:RDB文件检索工具

  • step4:配置⽂件，移动到/etc/⽬录下
    配置⽂件⽬录为解压文件中的 redis.conf 文件，将其移动至/etc/redis中

启动

服务器端

如果是通过包管理安装的话，可以使用service redis start进行启动。如果是通过源代码编译安装的话，则需要使用redis-server命令启动。

可以使⽤help查看帮助⽂档:redis-server --help

启动命令：redis-server /etc/redis/redis.conf

一般使用这条命令进行启动，可以指定加载的配置文件

如果要结束这个进程的时候，需要使用kill命令来结束进程:

  • ps -ef|grep redis：查看 redis 服务器进程 pid
  • kill -9 pid：杀死 redis 服务器进程

键命令

查找键

查找键的参数⽀持正则表达式

  • 查看所有键：keys *
  • 查看名称中包含 a 的键：keys 'a*'

判断键是否存在

如果存在返回1，不存在返回0

  • 判断键 a1 是否存在：exists a1

查看键对应的的类型

  • 查看键 a1 的值类型，为 redis ⽀持的五种类型中的⼀种：type a1

删除键及对应的值

  • 删除键 a2、a3：del a2 a3

设置过期时间

以秒为单位，如果没有指定过期时间则⼀直存在，直到使⽤ DEL 移除

  • 设置键 ‘a1’ 的过期时间为3秒：expire 'a1' 3

查看有效时间

以秒为单位

  • 查看键 ‘bb’ 的有效时间：ttl bb

string

字符串类型是 Redis 中最为基础的数据存储类型。它在 Redis 中是二进制安全的，这便意味着该类型可以接受任何格式的数据，如 JPEG 图像数据或 Json 对象描述信息等。在 Redis 中 string 类型的 Value 最多可以容纳的数据长度是 512M。

set

如果设置的键不存在则为添加，如果设置的键已经存在则修改

  • 设置键为 name 值为 abc 的数据：set name abc

setex

设置键值及过期时间，以秒为单位

  • 设置键为 aa 值为 aa 过期时间为 3 秒的数据：setex aa 3 aa

mset

设置多个键值

  • 设置键为 ‘a1’ 值为 ‘python’、键为 ‘a2’ 值为 ‘java’、键为 ‘a3’ 值为 ‘c’：mset a1 python a2 java a3 c

append

追加值

  • 向键为 a1 中追加值 ‘haha’：append 'a1' 'haha'

get

获取：根据键获取值，如果不存在此键则返回nil

  • 获取键 ‘name’ 的值：get 'name'

mget

根据多个键获取多个值

  • 获取键 a1、a2、a3 的值：mget a1 a2 a3

hash

hash ⽤于存储对象(结构体)，对象的结构为属性、值。值的类型 只能为string

hset

设置单个属性

注意：
如果使用hset命令的时候出现以下错误：

是由于Redis被配置为保存数据库快照，但它目前不能持久化到硬盘。于是用来修改集合数据的命令不能用。原因：强制关闭 Redis 快照导致不能持久化。

解决方案：运行 config set stop-writes-on-bgsave-error no命令后即可解决该问题。

  • 设置键 user 的属性 name 为 asdf：hset user name asdf

hmset

设置多个属性

  • 设置键 u2 的属性 name 为 asdf、属性 age 为 11：hmset u2 name asdf age 11

hkeys

获取指定键所有的属性

  • 获取键 u2 的所有属性：hkeys u2

hget

获取键属性

  • 获取键 u2 属性 ‘name’ 的值：hget u2 'name'

hmget

获取多个属性的值

  • 获取键 u2 属性 ‘name’、’age 的值：hmget u2 name age

hvals

获取所有属性的值

  • 获取键 ‘u2’ 所有属性的值：hvals u2

hdel

删除属性，属性对应的值会被⼀起删除

  • 删除键 ‘u2’ 的属性 ‘age’：hdel u2 age

如果要删除整个 hash 键及值，使⽤ del 命令

list

list类型是一种列表类型，列表的元素类型为 string

lpush

在左侧(列表头部)插⼊数据

  • 从键为 ‘a1’ 的列表左侧加⼊数据 a、 b、c：lpush a1 a b c

rpush

在右侧(列表尾部)插⼊数据

  • 从键为 ‘a1’ 的列表右侧加⼊数据 0、1：rpush a1 0 1

linsert

在指定元素的前或后插⼊新元素

  • 在键为 ‘a1’ 的列表中元素 ‘b’ 前加⼊ ‘3’：linsert a1 before b 3

lrange

返回列表⾥指定范围内的元素，start、stop 为元素的下标索引。索引从左侧开始，第⼀个元素为 0，索引可以是负数，表示从尾部开始计数，-1表示最后⼀个元素

  • 获取键为 ‘a1’ 的列表所有元素：lrange a1 0 -1

lset

设置指定索引位置的元素值

  • 修改键为 ‘a1’ 的列表中下标为 1 的元素值为 ‘z’：lset a 1 z

lrem

删除指定元素。将列表中前 count 次出现的值为 value 的元素移除

  • count > 0: 从头往尾移除
  • count < 0: 从尾往头移除
  • count = 0: 移除所有

如：

  • 向列表 ‘a2’ 中加⼊元素 ‘a’、’b’、’a’、’b’、’a’、’b’：lpush a2 a b a b a b
  • 从 ‘a2’ 列表右侧开始删除 2 个 ‘b’：lrem a2 -2 b
  • 查看列表 ‘a2’ 的所有元素：lrange a2 0 -1

set

set类型是一种⽆序集合，元素为string类型。元素具有唯⼀性，不重复。注意：对于集合没有修改操作。

sadd

添加元素

  • 向键 ‘a3’ 的集合中添加元素 ‘zhangsan’、’lisi’、’wangwu’：sadd a3 zhangsan sili wangwu

smembers

返回所有的元素

  • 获取键 ‘a3’ 的集合中所有元素：smembers a3

srem

删除指定元素

  • 删除键 ‘a3’ 的集合中元素 ‘wangwu’：srem a3 wangwu

zset

zset类型是一种有序集合，其元素为 string 类型，元素具有唯⼀性，不重复。每个元素都会关联⼀个 double 类型的 score，表示权重，通过权重将元素从⼩到⼤排序。

注意：没有修改操作

zadd

添加元素

  • 向键 ‘a4’ 的集合中添加元素 ‘lisi’、’wangwu’、’zhaoliu’、’zhangsan’，权重分别为 4、5、6、3：zadd a4 4 lisi 5 wangwu 6 zhaoliu 3 zhangsan

zrange

返回指定范围内的元素，start、stop为元素的下标索引，索引从左侧开始，第⼀个元素为0。索引可以是负数，表示从尾部开始计数，如 -1 表示最后⼀个元素。

  • 获取键 ‘a4’ 的集合中所有元素：zrange a4 0 -1

zrangebyscore

返回 score 值在 min 和 max 之间的成员

  • 获取键 ‘a4’ 的集合中权限值在 5 和 6 之间的成员：zrangebyscore a4 5 6

zscore

返回成员 member 的 score 值

  • 获取键 ‘a4’ 的集合中元素 ‘zhangsan’ 的权重：zscore a4 zhangsan

zrem

删除指定元素

  • 删除集合 ‘a4’ 中元素 ‘zhangsan’：zrem a4 zhangsan

zremrangebyscore

删除权重在指定范围的元素

  • 删除集合 ‘a4’ 中权限在 5、6 之间的元素：zremrangebyscore a4 5 6
