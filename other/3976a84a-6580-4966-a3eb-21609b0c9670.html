正则表达式

发表于2023-02-20|更新于2023-02-20|Other
|字数总计:1.2k|阅读时长:4分钟|阅读量:

正则表达式（REGEXP：Regular Expressions），由一类特殊字符及文本字符所编写的模式，其中有些字符（元字符）

正则表达式的元字符分类：字符匹配、匹配次数、位置锚定、分组

不表示字符字面意义，而表示控制或通配的功能，类似于增强版的通配符功能，但与通配符不同，通配符功能是用来处理文件名，而正则表达式是处理文本内容中字符

字符匹配

  • .：匹配任意单个字符，可以是一个汉字
  • []：匹配指定范围内的任意单个字符，示例：[abc]、[0-9]、[a-z]、[a-zA-Z]
  • [^]：匹配指定范围外的任意单个字符，^在中括号内表示取反的意思，示例：[^abc]
  • \w：匹配单词构成部分，等价于[_0-9a-zA-z]
  • \W：匹配非单词构成部分，等价于[^_0-9a-zA-z]
  • \S：匹配任何非空白字符。等价于[^ \f\n\r\t\v]
  • \s：匹配任何空白字符，包括空格、制表符、换页符等等。等价于[ \f\n\r\t\v]。注意 Unicode 正则表达式会匹配全角空格符

匹配次数

用在要指定次数的字符后面，用于指定前面的字符要出现的次数

  • *：匹配前面的字符任意次，包括0次，贪婪模式：尽可能长的匹配
  • .*：任意长度的任意字符
  • ? ：匹配其前面的字符出现0次或1次,即:可有可无
  • + ：匹配其前面的字符出现最少1次,即:肯定有且 >=1 次
  • {n}：匹配前面的字符n次
  • {m,n}：匹配前面的字符至少m次，至多n次
  • {,n}：匹配前面的字符至多n次,<=n
  • {n,}：匹配前面的字符至少n次

定位符

定位符可以用于定位出现的位置

  • ^：定位行首，用于模式的最左侧
  • $：定位行尾，用于模式的最右侧
  • ^PATTERN$：用于模式匹配整行
  • \< 或 \b：定位词首，用于单词模式的左侧
  • \> 或 \b：定位词尾，用于单词模式的右侧
  • \<PATTERN\>：匹配整个单词，注意：单词由字母数字和下划线组成

分组

分组：将多个字符捆绑在一起，当作一个整体处理，如：(root)+

后向引用：分组括号中的模式匹配到的内容会被正则表达式引擎记录于内部的变量中，这些变量的命名

正则表达式在其他环境中的使用

linux

linux 中正则表达式被很多程序和开发语言所广泛支持：vim,less,grep,sed,awk,nginx,mysql 等

查看 linux 正则表达式的帮助信息：man 7 regex

在 linux 中正则表达式分两类：

  • 基本正则表达式：BRE Basic Regular Expressions
  • 扩展正则表达式：ERE Extended Regular Expressions

linux 的扩展正则表达式和之前介绍的一样，而基本正则表达式则在某些地方中需要加上 \，需添加的地方有：

  • \?
  • \+
  • \{n\}
  • \{m,n\}
  • \{,n\}
  • \{n,\}
  • \<：词首
  • \>：词尾
  • \(\)：分组
  • \|：或者

另外，linux 中的正则表达式有些特有的内容，如下：

  • [:alnum:]：字母和数字，相当于 [A-Za-z0-9]
  • [:alpha:]：代表任何英文大小写字符，亦即 [A-Za-z]
  • [:lower:]：小写字母,示例,相当于 [a-z]
  • [:upper:]：大写字母，相当于 [A-Z]
  • [:blank:]：空白字符（空格和制表符）
  • [:space:]：包括空格、制表符(水平和垂直)、换行符、回车符等各种类型的空白,比 [:blank:] 包含的范围广
  • [:cntrl:]：不可打印的控制字符（退格、删除、警铃…）
  • [:digit:]：十进制数字
  • [:xdigit:]：十六进制数字
  • [:graph:]：可打印的非空白字符
  • [:print:]：可打印字符
  • [:punct:]：标点符号
