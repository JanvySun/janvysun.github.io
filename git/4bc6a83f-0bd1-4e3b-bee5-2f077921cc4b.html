本文介绍 Git 的常用命令

git init

在目录中创建新的 Git 本地仓库。

git clone

拷贝一个远程仓库到本地，可以使用 HTTP 或 SSH 的方式：

git add

将修改过的内容添加到暂存区中

git checkout

撤销工作区中编辑的内容，恢复到暂存区中保存的内容

ckeckout 也可以用于切换分支 (使用 -b 参数可以在分支不存在的情况下创建一个分支，创建的时候可以指定一个分支，默认为当前分支)

git status

查看在上次提交之后是否存在修改

git diff

git commit

将暂存区中的内容添加到仓库中。注意，Git 会为我们每一次提交记录名字和电子邮箱，因此我们需要在 ~/.gitconfig 中配置我们的用户名和邮箱

git log

该命令用于查看版本记录，包括版本号、提交记录、版本信息等

使用 --pretty=oneline 可以显示较为简单的信息，让每个版本都显示为一行；使用 --graph 可以显示分支的图形信息

git reset

该命令用于回退到以前的版本

使用版本指针时，HEAD 表示当前的版本，HEAD^ 表示上一个版本，HEAD^^ 表示当前版本的前前个版本；也可以使用 HEAD~1 表示上一个版本，HEAD~100 表示当前版本的前 100 个版本。
使用版本号时，给出版本号的前几位 (能区分每个版本) 即可。

我们也可以使用 reset 来撤销 add 的内容，如下：

git reflog

可以查看所有版本的所有操作记录 (包括已经被删除的 commit 记录和 reset 记录)

git rm

该命令用于删除一个文件，我们也可以使用 rm 命令来删除文件，删除之后用 add 和 commit 进行提交，但是使用 git rm 则不需要 add 操作。

git tag

该命令用于给版本打上一个标签。比如说，我们想为我们的商城项目发布一个「1.0.0-RELEASE」版本。我们可以用 git tag -a 1.0.0-RELEASE 命令给最新一次提交 (HEAD) 打上「1.0.0-RELEASE」的标签。

-a 选项意为 “创建一个带注解的标签”。 不用 -a 选项也可以执行的，但它不会记录这标签是啥时候打的，谁打的，也不会让你添加个标签的注解。推荐一直创建带注解的标签。
