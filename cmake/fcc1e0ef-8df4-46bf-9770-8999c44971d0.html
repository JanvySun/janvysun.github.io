CMake 的常用指令

CMAKE_MINIMUN_REQUIRED

指定 cmake 所需的最小版本

  • <PEOJECTNAME>_BINARY_DIR：编译路径，等同于 PROJECT_BINARY_DIR

SET

用于设置变量

变量设置之后，使用 ${VARNAME} 来读取变量

自带变量

  • CMAKE_CXX_STANDARD：指定工程的 C++ 标准，如 98、11 等
  • CMAKE_CXX_STANDARD_REQUIRED：BOOL类型，表示是否一定要支持指定的 C++ 标准，默认 OFF，通常情况下设置为 ON，更安全
  • CMAKE_CXX_EXTENSIONS：BOOL类型，默认为 ON，表示启用 GCC 特有的一些扩展功能，OFF 则表示关闭，为跨平台，通常情况下需要关闭
  • CMAKE_CXX_COMPILER：指定用哪个编译器，常见的有 g++、clang++、ninja 等
  • EXECUTABLE_OUTPUT_PATH：指定工程的可执行文件生成路径，如 ${PROJECT_BINARY_DIR}/bin
  • LIBRARY_OUTPUT_PATH：指定库的生成路径，如 ${PROJECT_BINARY_DIR}/lib
  • PROJECT_SOURCE_DIR：当前项目源码路径，存放 main.cpp 的地方
  • PROJECT_BINARY_DIR：当前项目输出路径，存放 main.exe 的地方
  • CMAKE_SOURCE_DIR：根项目源码路径
  • CMAKE_BINARY_DIR：根项目输出路径
  • CMAKE_MODULE_PATH：设置 CMake 模块路径，常用于查找第三方库
  • CMAKE_SYSTEM_NAME：CMake 所在系统的名字，如“Windows”、“Linux”等，可用于判断：
  • PROJECT_IS_TOP_LEVEL：BOOL类型，表示当前项目是否为根项目
  • PEOJECT_NAME：当前项目名
  • CMAKE_PROJECT_NAME：根项目的项目名
  • CMAKE_INSTALL_PREFIX：指定安装路径，默认为 /usr/local
  • CMAKE_BUILD_TYPE：指定构建类型，有如下值：
    Debug：调试模式，完全不优化，生成调试信息，方便调试程序
    Release：发布模式，优化程序最高，性能最佳，但是编译较慢
    MinSizeRel：最小体积发布，生成的文件比 Release 更小，不完全优化，减少二进制体积

补充

设置 <包名>_DIR 变量指向 <包名>Config.cmake 所在位置


MESSAGE

向终端输出用户自定义的信息，主要包含三种信息：

  • SEND_ERROR：产生错误，且生成过程将被跳过，会产生错误日志
  • STATUS：输出前缀为 – 的信息
  • FATAL_ERROR：产生错误，且立即终止所有 cmake 过程

ADD_EXECUTABLE

生成可执行文件

  • add_executable(EXE_NAME SOURCE_LIST...)：可执行文件名为 EXE_NAME，源文件为 SOURCE_LIST，可以是多个，也可以使用变量来定义

注：工程名与这里的可执行文件名没有关联


ADD_SUBDIRECTORY

这个指令用于向当前工程添加存放源文件的子目录，并可以指定中间二进制和目标二进制存放的位置，用法如下：

  • source_dir：指定子目录，将其加入工程中
  • binary_dir：指定编译输出(包含中间结果)路径，不指定，编译结果默认存放在 build/src 目录中
  • EXCLUDE_FROM_ALL 函数是将写的目录从编译中排除，如程序中的 example

ADD_LIBRARY

  • add_library(LIB_NAME SHARED SOURCE_FILE)：构建动态库，注意，动态库在可执行程序运行时，需要在其当前目录或者PATH目录中找到此动态库
  • add_library(LIB_NAME STATIC SOURCE_FILE)：构建静态库。静态库连接到可执行程序中去，会将库中的内容插到目标中(插桩)。
  • add_library(LIB_NAME OBJECT SOURCE_FILE)：构建对象库，对象库类似于静态库，但不生成 .a 文件，如果库与可执行程序在同一工程中，推荐使用对象库来代替静态库

注意：如果需要同时构建动态库与静态库，二者的 LIB 名不能相同，会报错，可以使用 SET_TARGET_PROPERTIES 来使其相同

注意：动态库无法链接静态库


SET_TARGET_PROPERTY

设置属性，与 set() 类似

如：

  • set_target_property(LIB_NAME PROPERTY OUTPUT_NAME "NEW_NAME")：设置某个 lib 的输出名
  • set_target_property(LIB_NAME PROPERTY CLEAN_DIRECT_OUTPUT 1)：
  • set_target_property(LIB_NAME PROPERTY VERSION # SOVERSION #)：设置库的版本

SET_TARGET_PROPERTIES

用来设置多个属性

注意：set 需要添加在 add_executable 之前，而 set_target_properties 需要在其之后


INCLUDE_DIRECTORIES

向工程添加头文件搜索路径 (会将这个搜索路径添加到整个工程中来，不能指定目标，不推荐使用，更推荐 target_include_directories)


ADD_DEFINITIONS

添加一个宏定义，会将该宏添加到每个目标上，不推荐，更推荐使用 target_add_definitions


ADD_COMPILE_OPTIONS

添加编译器命令行选项，会给所有目标都添加此选项，不推荐，更推荐使用 target_compile_options


TARGET_INCLUDE_DIRECTORIES

添加头文件搜索目录，效果与 include_directories 类似

常与 add_library 配合使用

此外，如果不希望让引用这个 lib 的可执行文件自动添加这个路径，可以将 PUBLIC 改为 PRIVATE 即可


LINK_DIRECTORIES

添加非标准的共享库搜索路径


TARGET_LINK_LIBRARIES

添加需要连接的库

注意：如果这个库不在系统的环境变量中，则需要先将库的路径添加到 link_directories 中

解决 Windows 编译后提示缺库的错误


TARGET_ADD_DEFINITIONS

添加一个宏定义


TARGET_COMPILE_DEFINITIONS

添加一个宏定义


TARGET_COMPILE_OPTIONS

添加编译器命令行选项


TARGET_SOURCES

添加要编译的源文件

如果不想在 add_executable 里添加所有的文件，就可以使用该指令（建议将 .h 文件也添加到里面来）


FILE

可以使用 GLOB 来自动查找当前目录下指定扩展名的文件，实现批量添加源文件，其中的 CONFIGURE_DEPENDS 是为了让 CMake 重新 build 的时候去检查有没有新文件添加

使用 GLOB_RECURSE 可递归搜索目录，需注意它会将 build 目录中的临时 cpp 文件也加进来，使用的时候需慎重


AUX_SOURCE_DIRECTORY

根据当前语言，自动查找目录下所需的文件，放到变量中，可与 target_sources 配合使用


FIND_PACKAGE

可以通过 find_package 命令寻找系统中的包或者库，然后通过 target_link_libraries 来链接

cmake 认为一个包（package）可以提供多个库，又称之为组件（components），如链接 TBB 的 tbb 与 tbbmalloc 组件：

其中的 find_package 会去 /usr/lib/cmake/TBB/TBBConfig.cmake 文件，然后的操作会由此文件来继续查找

find_package 用到了缓存机制，第一次执行 cmake -B build 时，会将查找的内容缓存起来，避免重复执行 cmake -B build 时速度变慢的问题


INSTALL

install 可用于安装程序

  • install(FILES 文件 DESTINATION 目录)
    其中的 目录 可以写绝对路径，也可以写相对路径，相对路径的实际路径是 ${CMAKE_INSTALL_PREFIX}/DESTINATION
  • install(PROGRAMS 文件 DESTINATION 目录)
    PROGRAMS 的文件是非目标文件的可执行程序，如脚本，通常放到 /usr/local/bin 下
  • install(DIRECTORY 目录 DESTINATION 目录)
    将 DIRECTORY 目录中的文件安装到指定目录中，通常用于一些文档的安装，如 install(DIRECTORY doc/ DESTINATION /USR/SHARE/DOC/TEST)
  • install(TARGETS 库名 LIBRARY DESTINATION 动态库目录 ARCHIVE DESTINATION 静态库目录)
    安装库的时候可以与 FILES 配置使用来安装头文件
  • install(TARGETS target [CONFIGURATIONS Debug] RUNTIME DESTINATION Debug/bin)
    install(TARGETS target [CONFIGURATIONS Release] RUNTIME DESTINATION Release/bin)
    安装目标文件可执行程序
