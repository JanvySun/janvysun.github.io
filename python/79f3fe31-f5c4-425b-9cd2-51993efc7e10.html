解释器

  • 计算机不能直接理解任何除机器语言以外的语言，所以必须要把程序员所写的程序语言翻译成机器语言，计算机才能执行程序。
  • 将其他语言翻译成机器语言的工具，被称为编译器。
  • 编译器翻译的方式有两种：一个是编译，另外一个是解释。
  • 两种方式之间的区别在于翻译时间点的不同。当编译器以解释方式运行的时候，也称之为解释器。

编译型语言

程序在执行之前需要一个专门的编译过程。把程序编译成为机器语言的文件，运行时不需要重新翻译，直接使用编译的结果就行了。程序执行效率高，依赖编译器，跨平台性差些。如 C、C++ 等。

解释型语言

解释型语言编写的程序不进行预先编译，以文本方式存储程序代码，会将代码一句一句直接运行。在发布程序时，看起来省了道编译工序，但是在运行程序的时候，必须先解释再运行。本文将介绍的 Python 就是一个解释型语言。



交互式 Python

交互式 Python 是直接在终端中运行解释器，而不输入要执行的文件名。

在 Linux 中直接输入 python 命令即可进入 Python 的交互式界面，然后输入 Python 的代码，会立即看到程序执行结果。在交互式界面中输入 exit() 即可退出，或者按 Ctrl+D 也能退出。



注释

注释的作用：使用用自己熟悉的语言，在程序中对某些代码进行标注说明，增强程序的可读性。Python 中通常有两种注释：

  • 单行注释：在 Python 中以 “#“ 号开头的文字都是注释。
  • 多行注释：如果希望编写的注释信息很多，一行无法显示，就可以使用多行注释。

中文问题

Python 2.x 默认是 不支持 中文的，而 Python 3.x 是默认支持中文的。

如果要使用 python 2.x 的版本支持中文，则要在 Python 文件的开头加上 #encoding=utf-8 即可让 Python2.x 支持中文

标准写法应该是 #-*- coding:utf-8 -*- ，当然也可以写成 #coding=utf-8
