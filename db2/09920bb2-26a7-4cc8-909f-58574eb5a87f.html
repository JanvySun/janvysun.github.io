常见错误

日志空间满

在删除大量数据的时候出现 SQL0964C 的错误，需要扩展日志空间，

db2 get db cfg for <db_name> | grep LOG 查看日志相关配置，主要是修改如下配置：

日志文件大小改为 4096 * 4k：db2 update db cfg for <DB_NAME> using LOGFILSIZ 4096

主日志文件个数改为 200：db2 update db cfg for <DB_NAME> using LOGPRIMARY 200

然后重启数据库
