用户操作

写一个脚本，完成以下任务：

  • 添加 1 个用户: user1
  • 每个用户的密码同用户名，而且要求，添加密码完成后不显示 passwd 命令的执行结果信息
  • 每个用户添加完成后，都要显示用户某某已经成功添加

基本逻辑

写一个脚本：如果用户 user 不存在，就添加用户 user

复杂逻辑

  • 如果用户存在，就显示用户已存在；否则，就添加此用户； id user1 && echo "user1 exists." || useradd user1
  • 如果用户不存在，就添加；否则，显示其已经存在； ! id user1 && useradd user1 || echo "user1 exists."
  • 如果用户不存在，添加并且给密码；否则，显示其已经存在； ! id user1 && useradd user1 && echo "user1" | passwd --stdin user1 || echo "user1 exists."

磁盘操作

写一个脚本partDisk.sh 假设已有硬盘为/dev/sdb, 为指定的硬盘创建分区：

  • 列出当前系统上所有的磁盘，让用户选择，如果选择quit则退出脚本；如果用户选择错误，就让用户重新选择；
  • 当用户选择后，提醒用户确认接下来的操作可能会损坏数据，并请用户确认；如果用户选择y就继续，n就退出；否则，让用户重新选择；
  • 抹除那块硬盘上的所有分区(提示，抹除所有分区后执行sync命令，并让脚本睡眠2秒钟后再分区)，并为其创建三个主分区，第一个为20M，第二个为512M, 第三个为128M，且第三个为swap分区类型

写一个脚本 syntax.sh，此脚本用于判定一个指定的 bash 脚本是否有语法错误。如果有错误，则提醒用户键入 Q 或者 q 无视错误退出，其他任何键可以通过 vim 打开这个指定脚本；如果用户通过vim打开编辑后保存退出时仍有错误，则重复第一步的内容；否则就正常关闭退出

写一个脚本service.sh，此脚本就是一个服务，放到 /etc/rc.d/init.d/ 目录下，然后用 chkconfig 命令来生成对应级别的链接文件

要求此服务在 2345 级别下启动，其他级别关闭，启动优先级为 77
